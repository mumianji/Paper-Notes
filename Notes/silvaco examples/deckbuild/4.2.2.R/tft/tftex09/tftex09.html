TFT Simulation of Grains and Grain Boundaries

Requires:S-Pisces/TFT
Minimum Versions:Atlas 5.26.1.R


This example demonstrates how Atlas and SPISCES may be used to model the influence of grain boundaries on the device simulation results. The input deck has been created in such a way as to allow easy implementation into a VWF automation tools environment.

The input deck initially uses the internal simulator within DeckBuild to define a number of variables using the DeckBuild set statement syntax. There are six variables the the user is required to specify which define the physical geometry of the TFT. The other variables are automatically defined and are used to create the mesh, regions and electrodes.

The grain and grain boundary are created by defining two regions in the polysilicon. Each region then has its own material and defect statements that define the properties within the grain and the grain boundary.

The remainder of the syntax is as normal and is described in the previous TFT examples.

To load and run this example, select the Load button in DeckBuild > Examples. This will copy the input file and any support files to your current working directory. Select the Run button in DeckBuild to execute the example.
